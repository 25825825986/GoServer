TCP通信协议

系统使用JSON格式的TCP协议进行通信，每个消息以换行符结束

1. 处理单条数据 (process)

// 请求
{"id": "1", "cmd": "process", "data": {"name": "order", "amount": 299.99}}

// 响应
{"id": "1", "status": "ok", "data": {"status": "processed"}, "latency": 5}

2. 批量处理 (batch)

// 请求
{"id": "2", "cmd": "batch", "data": [{"id": 1}, {"id": 2}, {"id": 3}]}

// 响应
{"id": "2", "status": "ok", "data": {"total": 3, "success_count": 3}}

3. 发布订阅 (publish/subscribe)

// 订阅频道
{"id": "3", "cmd": "subscribe", "channel": "news"}

// 发布消息
{"id": "4", "cmd": "publish", "channel": "news", "data": {"title": "Hello"}}

// 订阅者收到的推送
{"id": "pubsub", "status": "ok", "data": {"channel": "news", "data": {...}}}

4. 心跳检测 (ping)

// 客户端发送心跳（每30秒）
{"id": "5", "cmd": "ping"}

// 服务器响应
{"id": "5", "status": "ok", "data": {"type": "pong"}}

5. 获取指标 (metrics)

// 请求
{"id": "6", "cmd": "metrics"}

// 响应
{"id": "6", "status": "ok", "data": {
  "processed_count": 1000,
  "failed_count": 0,
  "avg_latency": 3.5
}}

测试工具

使用以下命令测试TCP服务器

1. 使用Telnet测试

# 连接服务器
$ telnet localhost 8080

# 发送测试数据
{"id":"1","cmd":"process","data":{"test":"value"}}

# 发送心跳
{"id":"2","cmd":"ping"}

2. 使用测试客户端

# 进入测试目录
$ cd test

# 单条测试
$ go run tcp_client.go -type=single

# 批量测试
$ go run tcp_client.go -type=batch

# 发布订阅测试
$ go run tcp_client.go -type=pubsub

# 压力测试（100连接 x 1000请求）
$ go run tcp_client.go -type=stress -c 100 -n 1000

3. 使用nc命令

# 发送单条数据
$ echo '{"id":"1","cmd":"process","data":{"name":"test"}}' | nc localhost 8080
